# RNG 增量动画格式

`RNG.MKF` 用于保存原版过场动画。当前实现以固定 SDLPal 基准的 `rngplay.c` 为行为参考，只支持本项目第一阶段目标所需的 DOS `YJ_1` 数据。

## 容器层级

1. `RNG.MKF` 外层分块表示一段动画。
2. 每段动画本身仍是 MKF 偏移表，内层分块表示压缩帧。
3. 每个非空帧先经 YJ1 解压，结果不是完整图片，而是一串增量绘制指令。
4. 解码器在 320×200、8-bit 索引画布上保留上一帧；指令可以跳过未变化像素、写入像素对或重复像素对。

因此不能把任意一帧脱离前序状态单独解码。运行时若从非零帧开始，`RngPlaybackStream` 会先从第 0 帧顺序预热到指定起始帧，再把恢复出的画布交给播放器。

## 已实现指令

| 操作码 | 含义 |
| --- | --- |
| `00`、`13` | 结束当前帧 |
| `02`–`04` | 按像素对跳过目标位置 |
| `06`–`0A` | 直接写入 1–5 个像素对 |
| `0B`、`0C` | 按 8-bit 或 16-bit 长度直接写入像素对 |
| `0D`–`10` | 重复同一像素对 2–5 次 |
| `11`、`12` | 按 8-bit 或 16-bit 长度重复同一像素对 |

解码器会检查源数据提前结束、未知操作码和目标画布越界。合成测试覆盖嵌套帧表、跳过、连续写入、重复写入、跨帧状态保留及损坏数据拒绝。

## 运行时流与版权边界

导入器默认只把原始压缩归档复制到被 Git 忽略的 `generated/pal/content/archives/rng.mkf`，并扫描帧表写入清单，不再生成或预加载整套 RGBA PNG。`RngPlaybackStream` 配置时只读取外层 MKF 偏移表，播放时才载入当前动画分块；YJ1 增量逐帧应用到一张 320×200 索引画布，再复用一个 RG8 `ImageTexture` 通过统一调色板 Shader 显示。停止播放后释放当前分块，因此内存不会随动画总帧数线性增长。

当前本地目标数据包含 12 段、1464 帧，其中剧情脚本通过 `0036` 引用 11 段、1410 帧。`ScriptVM 0036` 保存动画编号，`0037` 携带首尾帧和播放速度请求 `PalRngPlayer`。播放器使用当前剧情会话的日夜调色板，商标 RNG #6 则固定使用 SDLPal 指定的调色板 3；脚本在播放期间进入 `waiting_for_rng`，完成或资源损坏时都会异步解除等待，避免剧情永久卡住。片头、探索剧情和资源实验室浏览器复用同一套运行时流。

需要离线逐帧诊断时，可显式执行：

```bash
./tools/generate_resources.sh /path/to/Data --rng-previews
```

这会额外生成 `generated/pal/rng/{动画编号}/` PNG；下次默认导入会清除它们。压缩归档、可选 PNG、截图和其他原版派生内容都不能随项目代码提交或发布。仓库中只保留解码代码、中文格式文档和不含原版内容的合成测试数据。
