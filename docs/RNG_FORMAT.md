# RNG 增量动画格式

`RNG.MKF` 用于保存原版过场动画。当前实现以固定 SDLPal 基准的 `rngplay.c` 为行为参考，只支持本项目第一阶段目标所需的 DOS `YJ_1` 数据。

## 容器层级

1. `RNG.MKF` 外层分块表示一段动画。
2. 每段动画本身仍是 MKF 偏移表，内层分块表示压缩帧。
3. 每个非空帧先经 YJ1 解压，结果不是完整图片，而是一串增量绘制指令。
4. 解码器在 320×200、8-bit 索引画布上保留上一帧；指令可以跳过未变化像素、写入像素对或重复像素对。

因此不能把任意一帧脱离前序状态单独解码。运行时若从非零帧开始，还需要由调用方提供当时已有的画布状态，这与 SDLPal 的行为一致。

## 已实现指令

| 操作码 | 含义 |
| --- | --- |
| `00`、`13` | 结束当前帧 |
| `02`–`04` | 按像素对跳过目标位置 |
| `06`–`0A` | 直接写入 1–5 个像素对 |
| `0B`、`0C` | 按 8-bit 或 16-bit 长度直接写入像素对 |
| `0D`–`10` | 重复同一像素对 2–5 次 |
| `11`、`12` | 按 8-bit 或 16-bit 长度重复同一像素对 |

解码器会检查源数据提前结束、未知操作码和目标画布越界。合成测试覆盖嵌套帧表、跳过、连续写入、重复写入、跨帧状态保留及损坏数据拒绝。

## 本地预览与版权边界

导入器只转换首个可解码动画，按原版默认的每秒 16 帧输出到被 Git 忽略的 `generated/pal/rng/`，资源实验室可打开播放预览。预览暂用调色板 0；最终游戏会由脚本上下文决定实际调色板及淡入淡出。

这些 PNG 是原版素材的转换结果，不能随项目代码提交或发布。仓库中只保留解码代码、中文格式文档和不含原版内容的合成测试数据。
