# 经典战斗系统

Godot 版以固定 SDLPal 的经典回合制路径为行为基准，不启用 `ENABLE_REVISIED_BATTLE` 蓄力模式。静态资源、普攻回合、普通合击、角色保护、玩家/敌人攻击与状态仙术、基础使用/投掷物品、毒与九种经典状态、敌人三类战斗脚本、逃跑、主经验成长和 `ScriptVM 004A/0007` 剧情桥接已经接通。

## 敌队解析链路

脚本 `0007` 的第一个操作数是敌队编号，不是敌人编号。运行时按以下顺序解析：

```text
敌队编号
  → PalEnemyTeam.object_ids
  → PalEnemyObjectDefinition.enemy_id
  → PalEnemyDefinition + ABC.MKF Sprite
```

`0xFFFF` 和零槽位不生成敌人。有效对象按原顺序压紧，再用 `DATA.MKF #13` 的 `position[enemy_index][enemy_count - 1]` 取得脚底坐标；敌人属性中的 `y_position_offset` 最后叠加。玩家使用 `battle.c` 的一人、二人、三人固定站位表。

操作码 `004A` 只设置战场编号，真正启动战斗的是 `0007`。当前前期强制战斗数据为：

```text
6964: 004A [21, 0, 0]       # 战场 21
6965: 0007 [18, 40091, 0]  # 敌队 18，战败跳转 40091，不允许逃跑
```

## 当前战斗样板

资源实验室中的“战斗样板”默认加载敌队 18、战场 21 和李逍遥/赵灵儿：

- 主行动按官方菱形排列：上攻击、左仙术、右合击、下其他；
- 右侧合击仅在发动者和至少一名队友健康时亮起；单体合击继续选择敌人，全体合击直接提交；
- 选择攻击后，方向键切换仍存活的敌人，空格或回车确认；只剩一个敌人时直接确认；
- 选择仙术后显示角色已学会的真实仙术和“消耗 MP / 当前 MP”；单体治疗会选择我方角色，单体攻击会选择敌人；
- 选择下方“其他”后打开经典“自动／物品／防御／逃跑／状态”菜单；物品继续进入“使用／投掷”和 3×7 背包页；
- 主指令阶段按 `R` 会让当前及尚未选择指令的队员重复上一回合指令；第一回合按 `R` 默认全队普攻；
- `D` 为当前队员提交防御；
- `[` / `]` 切换非空敌队，`PageUp` / `PageDown` 切换战场背景；
- Esc 返回资源实验室。

独立样板没有前置剧情脚本可执行，因此显式使用当前 DOS 数据的标准战斗曲 RIX 37 并循环播放；切换敌队、重开战斗或从胜利结算返回时都会恢复 37。正式剧情战斗不使用该默认值，仍严格播放此前 `0045` 写入 `GameSession.battle_music_number` 的曲目。

样板使用新的临时 `GameSession`，不会污染探索进度，并为测试补满两名队员 HP/MP，使赵灵儿的五灵咒可以直接验证；剧情战斗仍使用正式会话中的真实数值。它已使用 `PalBattleController` 真实执行攻击、防御、仙术、敌人物理攻击、自动防御、体力扣减、死亡和胜负，不伪造结果。`PalBattleUI` 按 `uibattle.c` 固定坐标绘制角色状态框、头像、HP/MP 数字、当前角色箭头和四向图标，并固定在所有按脚底 Y 排序的战斗人物之上；仙术窗口等大面板不会被人物反向盖住。选敌时直接让敌人 Sprite 按调色板索引低四位 `+7` 闪烁，并在左上角用接近单个角色状态框宽度的紧凑面板临时显示目标名称、红色血条与真实当前／最大 HP；确认或取消目标后立即隐藏。敌人体力是便于试玩验证的 Godot 辅助信息，原版 SDLPal 不提供该常驻面板，也不会因此改变伤害或结算逻辑。

玩家物理攻击按 `fight.c::PAL_BattleShowPlayerAttackAnim()` 使用 F.MKF 的备战帧 7、接近帧 8 和攻击帧 9，再让敌人受击变色、显示蓝色上浮伤害并回到原战位；起手、暴击与武器命中分别读取 `PLAYERROLES` 的原版音效字段。敌人物理攻击同样读取敌人属性中的待机、施法和攻击帧数，以及攻击、动作和命中音效，接近目标后显示格挡或受击帧、伤害数字和归位过程；自动格挡与角色倒下分别使用角色保护、死亡音效。

`PLAYERROLES.covered_by` 指定每名角色的保护人。濒死、混乱、睡眠或定身目标抽中自动防御时，若保护人在队、存活、不濒死且没有上述异常，就由保护人移动到目标前方并使用保护帧/音效挡下本次攻击；异常目标找不到健康保护人时不能自己闪避。该行为是原版零伤害保护，不会把伤害转扣给保护人。

玩家仙术先按 `PAL_BattleShowPlayerPreMagicAnim()` 左上蓄势并切换施法帧，再按 DATA.MKF 中的速度、重复、偏移和图层字段播放 FIRE.MKF Sprite，并在生效帧播放仙术 VOC。当前已支持：

- 气疗术、观音咒等由 `001B/001C/001D` 成功脚本直接增减 HP/MP 的单体或全体恢复仙术；普通治疗不会复活倒地角色；
- 风、雷、冰、火、土等基础攻击仙术，使用 `PAL_CalcMagicDamage()` 的灵力随机修正、敌人防御、五灵/毒抗和战场五灵修正；
- 黄、青、蓝数字分别显示 HP 恢复、MP 恢复和伤害，结果继续写回同一 `GameSession`。

战斗效果解释器现支持 `0022` 复活、`0028–002C` 敌我下毒与按种类/等级解毒、`002D–002F` 敌我状态增减，以及指定毒/任意普通毒条件分支。净衣咒、金刚咒、回梦等脚本图完全落在已接入子集内时可以正常选择、消耗 MP 并生效；仍含召唤、变身或其他未实现战斗操作码的仙术保持不可用，不会只播放动画而漏掉效果。

## 经典合击

合击对象来自 `PLAYERROLES.cooperative_magic`，装备效果组 65 可以按官方顺序覆盖。经典模式要求发动者健康且全队至少有两名健康角色；“健康”表示存活、不濒死，并且没有混乱、定身、睡眠、封咒或傀儡。提交后不再询问后续队员，合击行动身法乘 10，执行后本回合其余玩家行动跳过，敌人行动仍正常保留；`R` 可以重复仍满足条件的上一回合合击。

每名贡献者消耗合击仙术 `mp_cost` 数值对应的 HP，最低保留 1 点，MP 不变。伤害强度为所有贡献者“武术 + 灵力”之和除以 4，再走该合击对象的基础伤害、敌人防御、五灵/毒抗和战场修正。画面使用 `fight.c` 的 `(208,157) / (234,170) / (260,183)` 多人站位、音效 29、角色施法帧和 FIRE 特效。当前真实资源中，李逍遥默认合击为对象 386“合体气功”，两人样板中每人消耗 9 HP；召唤类型合击仍等待召唤系统。

敌人每次行动先按 `magic_rate` 的 0–10 概率参数决定是否施法。纯攻击仙术支持普通单体、攻击全体、攻击整体和攻击战场四种类型；单体保留随机目标，其余类型作用于所有存活队员。敌方灵力为 `signed(基础灵力) + (等级 + 6) × 6`，随后复用 `PAL_CalcMagicDamage()`，并按原版传入 `100 + 玩家五灵/毒抗` 与抗性倍率 20。角色防御使伤害除以 2，1/3 魔法自动防御再给除数加 1。

画面按 `PAL_BattleShowEnemyMagicAnim()` 先让敌人向右下蓄势，播放敌人施法音效和施法帧，再以原版单体/三人/整体/战场锚点播放 FIRE Sprite；生效后显示蓝色伤害、角色受击变色和逐段后退。当前本地数据 85 个带敌术的敌人中，60 个基础攻击或已接入状态脚本敌术可完整结算；敌队 17 的敌术对象 312 已通过真实逻辑和 OpenGL 动画回归。其余包含召唤、变身或尚未实现脚本的 25 个敌术会明确提示尚未支持，不会退化成普攻或凭空扣血。

## 敌人战斗脚本上下文

OBJECT 中每个敌人保存 `script_on_turn_start`、`script_on_ready` 和 `script_on_battle_end` 三个入口。控制器会为每个单场 `EnemyState` 复制当前游标和可变仙术字段，不修改 `PalContentDatabase` 的静态定义：

- 开战前及每轮毒/状态结算之后运行 `script_on_turn_start`，可在玩家选择指令前播放对白、音效或改变敌人阶段；
- 敌人行动前运行 `script_on_ready`，随后使用脚本更新后的仙术和施法率执行本次 AI；
- 胜利领取奖励时逐敌运行 `script_on_battle_end`，随机掉落会直接进入 `GameSession.inventory`，配套居中消息继续由战斗 UI 播放。

当前专用解释器支持 `0000–0006` 流程、战斗内上下/下方/居中对白、音效与音乐、敌人减半体力、体力百分比分支、动态仙术、立即击倒、敌人逃跑、队伍成员条件、重复敌人判定、召唤、分裂、变身和随机分支。逻辑层把对白、音频、延迟、召唤/变身同步和逃跑封装为 `ScriptEvent`；`PalBattlePreview` 使用既有 `PalDialogBox`、统一音频播放器和敌人 Sprite 阵列播放，不在 UI 中重复计算数值。

真实 DOS 数据的静态敌队共引用 27 个回合开始入口、22 个行动准备入口和 10 个战后入口，所有可达操作码均由本地资源审计覆盖。敌队 22 已加入战斗内对白 OpenGL 截图，敌队 25 的脚本逃跑和敌队 46 两个对象各自掉落一件物品也通过真实运行验证；召唤会重新同步最多五个敌人的原版站位，`0069` 使用独立 `TERMINATED` 结果继续剧情而不冒充玩家逃跑或普通胜利。

## 战斗物品与逃跑

经典“其他”菜单按 `uibattle.c` 使用“自动、物品、防御、逃跑、状态”五项布局；当前物品、防御和逃跑可执行，自动战斗与状态页保持不可用色。物品子菜单继续分为“使用／投掷”，背包页复用原版 3×7 网格、数量、BALL 图标和说明。选择单体恢复品后指向我方角色，投掷暗器则进入敌人闪烁选择；全体物品跳过单体选择。

控制器在指令阶段预留会消耗的物品数量，避免两名队员同时提交背包里的最后一个物品。执行时才真正修改 `GameSession.inventory`：

- 使用物品支持由 `001B/001C/001D` 增减 HP/MP，并复用战斗效果解释器执行复活、毒、解毒和状态脚本；播放原版物品名称、角色帧 5、音效 28、目标变色和黄／青恢复数字；
- 投掷物品支持 `0042` 模拟仙术、`0021` 固定伤害和 `0066` 武器伤害，使用 FIRE 特效、角色帧 5/6、原版投掷音效和蓝色伤害数字；
- 含隐身、变身或尚未实现条件操作码的特殊物品仍显示为不可用，不会只扣库存而漏掉效果。

本地 DOS 数据现有 68 个使用物品与 49 个投掷物品脚本可以走当前完整结算路径；止血草和梅花镖已通过真实脚本、库存和 OpenGL 动画回归。

逃跑按 `fight.c` 对齐：当前角色逃跑值与 `RandomLong(0, 全部存活敌人的身法 + (等级 + 6) × 4)` 比较。经典全队逃跑键会让尚未选指令的队员都尝试逃跑；成功时播放音效 45 并让全队退出战场，结果值 `FLED = 3` 直接进入 `0007 operand[2]` 分支。Boss 战仍允许选择逃跑，但必定播放失败动作和“逃跑失败”，不会误离开剧情战斗。

## 当前回合逻辑

`PalBattleController` 不依赖任何 Godot 场景节点。它持有敌人本场体力、玩家指令、防御状态、物品预留和行动队列；玩家当前 HP/MP 与背包继续由 `GameSession` 持有。`ActionResult` 携带仙术/物品对象、目标、MP 消耗、伤害、恢复量和逃跑结果，`PalBattlePreview` 据此播放 Sprite、受击变色和数字，不重复计算数值。

全队完成指令后，敌人与玩家统一进入行动队列：

- 敌人身法为 `(等级 + 6) × 3 + signed(基础身法)`；
- 双动敌人加入两次，身法较低的一项标记为第二动；
- 合击行动把发动者身法乘 10，并记录本轮健康贡献者；
- 防御指令将玩家身法乘 5；
- 加速把玩家身法乘 3，勇气让物理攻击必定暴击，双击让单体/全体普攻各执行两次；
- 混乱角色攻击队友，混乱敌人攻击同伴；睡眠与定身跳过行动，封咒禁止选择仙术；
- 防护把角色承受的物理和仙术伤害减半；傀儡可让倒下角色在仍有存活队友时自动普攻；
- 濒死角色身法减半；
- 每项最终乘 `0.9–1.1`，再按身法从高到低执行。

普通行动队列结束后不会立即进入下一轮，而是按 `fight.c` 的经典顺序生成一项独立的回合末结果：先依次执行每名玩家所中毒的递进脚本并递减其九种状态，再执行每个敌人的毒脚本并递减敌人状态，之后才检查毒杀胜利、全灭或进入下一回合。毒脚本游标随角色/敌人保存，`0000` 会在下回合重复同一效果，`0001` 会推进到下一段；战斗结束只清除临时状态，中毒继续保留在 `GameSession`。画面会显示“毒性发作”、受击色偏和对应的伤害、HP/MP 恢复数字。

敌人物理攻击若配置 `attack_equivalent_item`，会在未自动防御时按 DATA 概率与玩家毒抗执行其物品脚本，因此带毒爪击不再只有普通攻击伤害。装备脚本维持的双击等状态单独保存在装备效果槽，不会被回合递减或普通解咒误删。

基础伤害忠于 `fight.c::PAL_CalcBaseDamage()`：

```text
攻击 > 防御       → 攻击 × 2 - 防御 × 1.6
攻击 > 防御 × 0.6 → 攻击 - 防御 × 0.6
否则              → 0
```

物理伤害再除以目标物理抗性。玩家单体普攻会加入 `1–2` 浮动、经典暴击和李逍遥额外一击判定；敌人物理攻击加入等级攻击、随机修正和 7/17 自动防御判定。最终有效普攻至少造成 1 点伤害。

敌人基础攻击、防御等字段来自 16 位 `WORD`。官方会在同一 16 位变量中加入等级修正，因此必须保留回绕语义：例如绿叶小妖的基础防御 `0xFFFA` 加上 `(0 + 6) × 4` 后为 `18`，不能由 Godot 的宽整数误算成 `65554`。玩家属性也会把六槽装备效果逐次按 `WORD` 相加；李逍遥的木剑等初始装备现已在首战前生效，对绿叶小妖的固定回归普攻由裸装最低伤害恢复为 40 点。

攻击仙术先把角色灵力乘 `RandomFloat(10, 11) / 10`，用同一基础伤害公式减去敌人防御后除以 4，再加仙术基础伤害；随后依次应用目标五灵/毒抗和当前战场的五灵修正，最低造成 1 点伤害。

固定随机种子使用移植自 `util.c` 的 32 位 LCG。这样测试可以精确复现行动顺序和伤害，同时正式运行仍会使用当前时间生成种子。

### 重复上一回合指令

`R` 对齐 `input.c::kKeyRepeat`、`fight.c::PAL_BattleCommitAction()` 和经典回合的 `fRepeat` 行为。上一回合缓存按队伍位置分别保存动作、仙术/物品编号和目标；重复回合的临时降级不会覆盖缓存。因此真气或库存后来恢复时，再按 `R` 仍会尝试原来的仙术或物品。

- 尚无上一回合缓存时，原版的空 `Pass` 指令转为普通攻击；
- 攻击仙术真气不足时改为普通攻击，恢复或辅助仙术真气不足时改为防御；
- 投掷物耗尽时改为普通攻击，使用物品耗尽时改为防御；
- 已倒下的敌方目标会从原目标位置开始重选，群体指令继续保持群体目标；
- 如果先为部分队员手动选择，再按 `R`，只为当前及后续尚未选择指令的队员复用缓存。

## 战后奖励与成长

敌人 HP 第一次降到零时，控制器按 `fight.c::PAL_BattlePostActionCheck()` 累计其经验和金钱；胜利后 `claim_victory_rewards()` 只允许领取一次，重复确认不会重复加钱或升级。结算顺序对齐 `battle.c::PAL_BattleWon()` 的主成长路径：

1. 金钱加入 `GameSession.cash`；
2. 战斗结束时仍存活的队员获得本场全部主经验，倒下角色跳过；
3. 按 `DATA.MKF #14` 当前等级阈值逐级扣除经验；
4. 每升一级随机增加最大 HP `10–17`、最大 MP `8–13`、攻击/灵力 `4–5`、防御/身法 `2–3`、逃跑值 `2`，并把 HP/MP 回满；
5. 按 `DATA.MKF #6` 学会当前等级已满足且尚未拥有的仙术；
6. 全队按经典模式恢复“当前值到上限差额”的一半，倒下角色也会因此恢复。

普通胜利播放 RIX 3，Boss 胜利播放 RIX 2。奖励总览使用原版“获得经验值／打败敌人得文钱”双窗口；升级页显示等级、体力、真气、武术、灵力、防御、身法和吉运的前后值。真实首战敌队 18 固定结算为 52 经验、96 文，李逍遥从 1 级升到 2 级。

敌人的 `script_on_battle_end` 现在会在专用战斗上下文中执行；`001F` 的零数量按原版 `PAL_AddItemToInventory()` 解释为获得一件，并限制单项最多 99。原版隐藏经验（按本场攻击、防御、施法等次数分配的七类成长）仍待后续阶段接入。

## 剧情战斗桥接

`004A` 把战场编号写入 `GameSession.battlefield_number`。执行 `0007` 时，VM 保存战败和逃跑分支，进入 `waiting_for_battle` 并向 `MapExplorer` 请求敌队、战场和 Boss 标志。探索输入、菜单、接触事件和 10 FPS 自动脚本在等待期间全部暂停。

`MapExplorer` 在同一个 HUD CanvasLayer 上覆盖剧情模式的 `PalBattlePreview`，直接复用探索的内容数据库与 `GameSession`，因此战斗扣除的 HP 不会在返回地图后丢失。进入时切换 `0045` 指定的战斗 BGM，确认胜负后恢复场景 BGM，再按以下规则调用 `complete_battle()`：

- 胜利继续 `0007` 的下一条指令；
- 战败且 `operand[1]` 非零时跳到该入口；
- 逃跑且 `operand[2]` 非零时跳到该入口；
- `operand[2] == 0` 同时表示 Boss 战、不允许逃跑。

战斗 UI 的逃跑成功结果已经直接使用 `ScriptVM.BATTLE_RESULT_FLED = 3`，普通剧情战会跳到原版逃跑入口；`operand[2] == 0` 的 Boss 战由控制器强制失败。若战斗资源缺失，探索层会显示具体原因并延后按战败分支恢复，避免在信号回调中重入脚本解释循环。
