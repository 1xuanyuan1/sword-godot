# 文档语言约定

- 项目说明、开发指南、架构、测试、里程碑和问题排查默认使用中文。
- 英文可以作为补充摘要，但不应替代中文主文档。
- GNU GPL 等许可证原文、第三方版权姓名、代码标识符、命令和外部接口名称保持原文，以免产生法律或技术歧义。
- 新增公开接口时必须提供中文用途说明；代码标识符和上游函数名保持原文。

## GDScript 注释规范

- 每个脚本在许可证头之后说明模块职责、输入输出和状态所有权。
- `class_name`、公开信号、公开属性以及不以下划线开头的函数使用 Godot `##` 文档注释。
- 公共函数注释应说明参数、返回值、失败方式，以及是否修改 `GameSession`、事件对象或生成目录。
- 私有函数只在算法、数据布局、坐标换算、兼容分支或状态机原因不直观时添加 `#` 注释。
- 从 SDLPal 改写的行为标明参考文件或函数；统一的固定提交记录在 [SDLPal 上游基准](UPSTREAM.md)。
- 不添加“给变量赋值”“遍历数组”等只是重复代码表面的注释。
- 大型控制器使用 `#region` 按职责分区，使 Godot 编辑器能够折叠浏览。

## 文档同步规则

- 新增核心目录或模块时，更新[项目目录结构](PROJECT_STRUCTURE.md)和[整体架构](ARCHITECTURE.md)。
- 修改运行、导入、测试或提交方式时，更新[开发工作流](DEVELOPMENT_WORKFLOW.md)。
- 修改地图、脚本、菜单或资源格式时，更新相应专题文档。
- 文档、注释、实现和测试应在同一功能里程碑内完成；缺少其中任一项时功能不进入完成状态。
