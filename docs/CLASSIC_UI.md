# 经典菜单与物品页

Godot 版菜单以固定 SDLPal 镜像的 `uigame.c`、`itemmenu.c` 和 `ui.c` 为行为及坐标基准，不再使用现代整屏面板和大按钮。

## 主菜单

- 逻辑画布保持 320×200，不增加全屏遮罩。
- 金钱窗位于 `(0, 0)`，使用官方单行窗口 Sprite 44–46；标签位于 `(10, 10)`，金额使用黄色数字 Sprite。
- 主菜单窗位于 `(3, 37)`，四项文字依次位于 `(16, 50)`、`(16, 68)`、`(16, 86)`、`(16, 104)`。
- 菜单文字直接读取原版词库中的“状态、仙术、物品、系统”，选中项使用官方调色板 `0xF9–0xFE` 循环色。
- 尚未实现的状态、仙术和系统功能暂以官方“不可用”颜色显示，不添加“后续”等非原版字样。

## 物品页

- 主菜单选择“物品”后显示原版“装备／使用”小窗；装备系统未完成前保持不可用，“使用”可以进入物品页。
- 物品页使用 `DATA.MKF #9` 的第二套九宫格窗口，布局为 3 列×7 行，行距 18、列距 100。
- 光标使用 UI Sprite 69；数量大于 1 时使用青色数字 Sprite 56–65。
- 左下物品框使用 UI Sprite 70，图标从本地 `BALL.MKF` 分块解码。
- 选中物品的说明从本地 `desc.dat` 读取，自 `(75, 150)` 起显示；原文中的 `*` 按官方规则换行，每行间隔 16 像素。
- Esc 返回上一级；方向键选择，空格或回车确认。`I` 快捷键仍可直接进入物品页。

## 资源边界

原版 UI Sprite、字库图集和物品位图只生成到被 Git 忽略的 `generated/pal/`。GitHub 仅保存读取、解码、布局与测试代码，不包含原版资源。
