# 经典菜单、状态、仙术与物品页

Godot 版菜单以固定 SDLPal 镜像的 `uigame.c`、`itemmenu.c` 和 `ui.c` 为行为及坐标基准，不再使用现代整屏面板和大按钮。

## 主菜单

- 逻辑画布保持 320×200，不增加全屏遮罩。
- 金钱窗位于 `(0, 0)`，使用官方单行窗口 Sprite 44–46；标签位于 `(10, 10)`，金额使用黄色数字 Sprite。
- 主菜单窗位于 `(3, 37)`，四项文字依次位于 `(16, 50)`、`(16, 68)`、`(16, 86)`、`(16, 104)`。
- 菜单文字直接读取原版词库中的“状态、仙术、物品、系统”，选中项使用官方调色板 `0xF9–0xFE` 循环色。
- 状态、仙术、物品和系统均已接通；主菜单不再保留不可用占位项。

## 状态页

- 使用 `FBP.MKF #0` 的原版 320×200 背景，并按 `palcfg.c` 默认布局绘制角色名、RGM 头像、等级、当前/下级经验、HP/MP 和武术、灵力、防御、身法、吉运。
- 六件装备使用 BALL 图标和对象名称放回原版环形位置；三级以下的当前毒状态显示在头像右侧。
- 左/右/上/下切换队员，空格或回车切到下一名队员，Esc 返回主菜单。

## 场外仙术页

- 多人队伍先在 `(35, 62)` 的原版小窗选择仍存活的施法者；单人队伍直接进入仙术列表。
- 仙术只从 PLAYERROLES 初始仙术和升级习得结果读取，不遍历 OBJECT 共用表，因此物品不会被误识别为仙术。
- 列表使用官方 3 列×5 行窗口、循环选中色和 UI Sprite 69 光标；只启用带 `kMagicFlagUsableOutsideBattle` 且当前 MP 足够的仙术。
- 有 DESC.DAT 说明时，左上角只显示“所需 MP / 当前 MP”，说明从 `(102, 3)` 开始；无说明时左上显示金钱、右上显示 MP，两个布局不会互相覆盖。
- 单体仙术使用底部角色状态框和向上箭头选择目标；全体仙术直接执行。`PalGameMenu` 只发出对象、施法者和目标，`MapExplorer` 依次运行 `script_on_use` 与 `script_on_success`，两段都成功后才扣除 MP 并返回仙术列表。

## 系统与音量

- 系统窗沿用官方 `(40, 60)` 位置和“储存进度、读取进度、音乐、音效、结束游戏”五行顺序。
- 储存／读取已接入 100 个独立 Godot 槽位，每页显示 5 个；上／下选择，左／右翻页，槽位摘要显示中文地点、时间、队伍头像、姓名和等级。
- 空槽、文件损坏、格式版本不兼容和 PAL 内容指纹不匹配使用不同状态；成功后显示居中系统 Toast。退出仍保持不可用颜色。
- 每行右侧使用原版黄色数字 Sprite 显示 0–100；左/右键以 10 为步长调整，空格/回车在静音和满音量间快速切换。
- 音量修改立即作用于 `PalAudioPlayer`，不会暂停或重启当前 BGM。

存档格式、保存范围和本机路径见[Godot 版本化存档系统](SAVE_SYSTEM.md)。

## 物品页

- 主菜单选择“物品”后显示原版“装备／使用”小窗；两项均已可用，并按用途过滤物品页。
- 物品页使用 `DATA.MKF #9` 的第二套九宫格窗口，布局为 3 列×7 行，行距 18、列距 100。
- 光标使用 UI Sprite 69；数量大于 1 时使用青色数字 Sprite 56–65。
- 左下物品框使用 UI Sprite 70，图标从本地 `BALL.MKF` 分块解码。
- 选中物品的说明从本地 `desc.dat` 读取，自 `(75, 150)` 起显示；原文中的 `*` 按官方规则换行，每行间隔 16 像素。
- Esc 返回上一级；方向键选择，空格或回车确认。`I` 快捷键仍可直接进入物品页。
- 固定 SDLPal 当前菜单代码没有通用 VOC 反馈；打开、移动选择和确认不再猜测短音效编号，避免与剧情或战斗音效错配。

## 装备页

- “装备”物品页只显示背包中的可装备物品；角色许可继续读取对象 flags，而不是按名字猜测。
- 选择物品后切换到 `FBP.MKF #1` 原版 320×200 背景，绘制 BALL 图标、数量、队员列表、六槽当前装备和五项实时属性。
- 确认换装会执行原版装备脚本；新物品离开背包、旧物品返回背包，所有属性立即刷新。
- 装备页坐标和 `wLastUnequippedItem` 连续换装行为详见[装备系统](EQUIPMENT.md)。

## 资源边界

原版 UI Sprite、状态 FBP、RGM 头像、字库图集和物品位图只生成到被 Git 忽略的 `generated/pal/`。GitHub 仅保存读取、解码、布局与测试代码，不包含原版资源。
