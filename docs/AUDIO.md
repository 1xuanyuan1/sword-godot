# 音乐、音效与音量

Godot 运行时不直接解释 RIX/OPL，也不直接播放 VOC。导入器在本机把两类原版数据转换为 Godot 可播放的 WAV；转换结果全部位于被 Git 忽略的 `generated/pal/audio/`。

## 离线导入

- `VOC.MKF`：`VocDecoder` 解码 8 位单声道 PCM，输出到 `generated/pal/audio/voc/NNN.wav`。
- `MUS.MKF`：GPL C++ 辅助工具复用固定 SDLPal 的 AdPlug RIX 播放器和 MAME OPL2 核心，输出 44.1 kHz 单声道 WAV 到 `generated/pal/audio/rix/NNN.wav`。
- RIX 不盲目导出全部分块。导入器扫描 `scripts.bin` 中的 `0x0043` 场景音乐、`0x0045` 战斗音乐和 `0x00A3 op1` CD 失败回退音乐，只生成剧情实际引用的曲目，并额外保留 `4/5` 作为启动/资源预览曲。
- 当前本地数据会生成 72 首有效曲目，约 289 MB；这些统计只描述本机生成结果，不会把原版音频提交到仓库。

## 运行时关系

`PalAudioPlayer` 使用一个 `AudioStreamPlayer` 播放 BGM，使用八个循环复用的声道允许短音效重叠。Godot 的 WAV 导入器在源文件没有循环标记时会给出 `loop_end = 0`；运行时开启循环的同时必须把循环范围显式设为 `0..采样总帧数`，否则播放器会在第 0 帧直接结束。`ScriptVM` 按官方 `script.c` 发出请求：

| 操作码 | 运行时行为 |
| --- | --- |
| `0x0043` | 切换场景 BGM；`op0` 是曲目，`op1 != 1` 表示循环，`op1 == 3` 对大多数曲目使用 3 秒淡入淡出 |
| `0x0045` | 保存下一场战斗的 BGM 编号，不立即切换场景音乐 |
| `0x0047` | 按 `op0` 播放一次对应的 VOC 音效 |
| `0x00A3` | 原版优先播放 `op0` CD 音轨；Godot 无 CD 设备，按官方回退播放 `op1` 的循环 RIX |

BGM 属于剧情场景状态，不按 `map_number` 硬编码。多个剧情场景可以复用同一地图却播放不同音乐；如果新场景没有执行 `0x0043/0x00A3`，当前曲目会自然续播。当前新游戏首场景由进入脚本请求曲目 `31`，同一段剧情会请求音效 `98`；比武招亲场景使用 `00A3`，桌面版实际回退到 RIX 14。

资源实验室的独立战斗样板没有前置剧情脚本，因此固定循环播放本数据版本的标准战斗曲 RIX 37；正式剧情战斗始终使用 `0x0045` 保存的编号，不按敌队或战场猜测音乐。普通/Boss 胜利页切换到 RIX 3/2，样板重开后再恢复 37。

### 遇敌切换与音效

固定 SDLPal 的 `battle.c::PAL_BattleMain()` 在进入战斗时依次停止场景音乐、等待 200 ms、执行 `VIDEO_SwitchScreen(5)` 像素切屏，再循环播放 `wNumBattleMusic`。这条通用路径没有调用 `AUDIO_PlaySound()`；本机 DOS 脚本表的随机遇敌入口同样是 `0050` 渐隐后执行 `0007`，没有额外的 `0047`。因此当前基准不存在一个可统一套用的“遇敌 VOC 编号”，不能把任意短音效猜成原版行为。特定剧情若在战前显式执行 `0047`，仍按脚本播放；通用遭遇的听觉反馈来自战斗 BGM 在切屏后的起奏。

## 探索与菜单反馈

固定 SDLPal 当前版本的通用地图移动和菜单代码没有为这些动作指定 VOC。Godot 版不再用 `VOC 1/2/3` 猜测脚步、打开、移动选择或确认音，避免把无依据的高音量反馈误当成原版行为。运行时只播放剧情脚本通过 `0x0047` 明确请求的音效；后续若从固定上游代码确认其他声音关系，再以相同方式补入测试。

## 系统菜单音量

`GameSession.music_volume` 与 `sound_volume` 分别保存 0–100 的百分比，默认均为 100。经典“系统”菜单沿用官方 `(40,60)` 五行窗口：

- “音乐”或“音效”行按左/右键以 10 为步长独立调整；
- 按空格/回车在 0 与 100 之间快速切换，保留原版开/关操作习惯；
- 调整后立即更新当前 BGM 与全部音效声道；
- 音乐和音效百分比随当前 Godot 存档保存；读档后立即重新应用到 BGM 与音效声道。

## 验证

```bash
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . \
  --script res://tests/run_local_audio_test.gd
```

该测试检查编号、资源存在性、完整 WAV 循环区间、持续播放状态，并用 `AudioEffectCapture` 验证 BGM 31、战斗 BGM 37 和 VOC 98 确实在 Master 总线产生非静音波形；不会转储或提交原版音频。实际试听从新游戏进入第一个场景，然后打开“系统”页分别调节音乐和音效。剧情测试因跳过场景进入脚本，检查点必须显式保存对应剧情时点的曲目编号，不能按地图编号猜测。
