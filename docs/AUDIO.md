# 音乐、音效与音量

Godot 运行时不直接解释 RIX/OPL，也不直接播放 VOC。导入器在本机把两类原版数据转换为 Godot 可播放的 WAV；转换结果全部位于被 Git 忽略的 `generated/pal/audio/`。

## 离线导入

- `VOC.MKF`：`VocDecoder` 解码 8 位单声道 PCM，输出到 `generated/pal/audio/voc/NNN.wav`。
- `MUS.MKF`：GPL C++ 辅助工具复用固定 SDLPal 的 AdPlug RIX 播放器和 MAME OPL2 核心，输出 44.1 kHz 单声道 WAV 到 `generated/pal/audio/rix/NNN.wav`。
- RIX 不盲目导出全部分块。导入器扫描 `scripts.bin` 中的 `0x0043` 场景音乐和 `0x0045` 战斗音乐，只生成剧情实际引用的曲目，并额外保留 `4/5` 作为启动/资源预览曲。
- 当前本地数据会生成 72 首有效曲目，约 289 MB；这些统计只描述本机生成结果，不会把原版音频提交到仓库。

## 运行时关系

`PalAudioPlayer` 使用一个 `AudioStreamPlayer` 播放 BGM，使用八个循环复用的声道允许短音效重叠。`ScriptVM` 按官方 `script.c` 发出请求：

| 操作码 | 运行时行为 |
| --- | --- |
| `0x0043` | 切换场景 BGM；`op0` 是曲目，`op1 != 1` 表示循环，`op1 == 3` 对大多数曲目使用 3 秒淡入淡出 |
| `0x0045` | 保存下一场战斗的 BGM 编号，不立即切换场景音乐 |
| `0x0047` | 按 `op0` 播放一次对应的 VOC 音效 |

BGM 属于剧情场景状态，不按 `map_number` 硬编码。多个剧情场景可以复用同一地图却播放不同音乐；如果新场景没有执行 `0x0043`，当前曲目会自然续播。当前新游戏首场景由进入脚本请求曲目 `31`，同一段剧情会请求音效 `98`。

## 菜单和移动反馈

固定 SDLPal 当前版本的通用菜单和地图移动代码没有主动播放反馈音，但玩家试玩需要开菜单、选择和脚步反馈。当前编号集中定义在 `PalAudioPlayer`：菜单打开 `1`、移动选择 `2`、确认 `3`、脚步 `2`。脚步每两个 half 格播放一次，避免 10 Hz 移动循环重复重启短 VOC。

这组反馈映射与剧情 `0x0047` 完全分开，后续按 DOS 实机试听校正时只修改集中常量，不会改剧情脚本数据。

## 系统菜单音量

`GameSession.music_volume` 与 `sound_volume` 分别保存 0–100 的百分比，默认均为 100。经典“系统”菜单沿用官方 `(40,60)` 五行窗口：

- “音乐”或“音效”行按左/右键以 10 为步长独立调整；
- 按空格/回车在 0 与 100 之间快速切换，保留原版开/关操作习惯；
- 调整后立即更新当前 BGM 与全部音效声道；
- 当前只保存在本次 `GameSession`，正式存档系统完成后再写入用户配置。

## 验证

```bash
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . \
  --script res://tests/run_local_audio_test.gd
```

该测试只检查编号、资源存在性、Godot 声道和音量状态，不转储或提交原版音频。实际试听从新游戏进入第一个场景，然后打开“系统”页分别调节音乐和音效。
