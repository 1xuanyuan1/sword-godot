# 脚本虚拟机与对话流程

Godot 版 `ScriptVM` 以固定 SDLPal 基准的 `script.c` 为行为参考。当前阶段优先覆盖首场景进入流程和附近事件实际使用的指令，尚未宣称完成原版全部操作码。

## 执行模型

- 解释器保存当前指令、事件对象编号和脚本调用栈。
- `0004` 调用的子脚本结束后会返回父脚本继续执行。
- `FFFF` 消息中的“角色名：”会识别为说话人标题，不单独等待；紧随其后的正文显示后，解释器暂停，玩家按空格或回车继续。
- 上、下、中部对话位置由 `003B`–`003E` 设置；脚本结束或重绘时关闭当前对话框。
- `003C`、`003D` 传入的 `RGM.MKF` 肖像编号会显示在左侧角色栏，姓名位于头像下方，正文位于右侧；无肖像旁白自动扩展正文区域。
- 对于脚本省略肖像、但保留“角色名：”的台词，内容数据库会从完整脚本统计该角色最常使用的肖像作为回退；明确的无标题角色台词使用单独元数据，避免误伤旁白。
- `003E` 系统消息使用中央黑底白字 Toast，约 1.4 秒后自动关闭，不需要按键。
- `003B` 普通中央对白保持交互式显示；以成对引号标记的无角色剧情叙述复用系统 Toast，连续消息会先合并，再统一计时关闭。
- 触发事件时传入全局 `EventObject` 编号，使方向、帧、位置、状态和层级修改作用于正确对象。
- 搜索事件使用空格/回车；门、楼梯和传送点按照 `trigger_mode 4–8` 在接近时自动触发。

## EventObject 自动脚本

场景事件以 10 FPS 调用 `PAL_RunAutoScript` 兼容层。普通指令每帧执行一条；`0003` 无条件跳转和 `0006` 概率跳转按官方 `goto begin` 在同一帧继续目标指令，避免 NPC 路线无故多停一帧。

当前自动脚本已支持等待、概率分支、子脚本、四方向单步、不同速度走到目标、方向/动作、修改目标脚本与状态、剧情音效、临时隐藏、追逐队伍、直接移动、逻辑层、区域判断和原地动画。追逐使用当前 `PalMapData` 与 EventObject 阻挡，浮空事件按脚本参数忽略阻挡。

事件消失状态对齐 `play.c`：负 `vanish_time` 计到零后直接重现；`0052` 会先把状态变负，正计时归零后仍保持隐藏，直到对象离开 SDLPal 的 320×320 镜头保护区才恢复正状态和第 0 帧。自动移动的 NPC 进入接触范围时，同一更新周期会触发其接触脚本。

## 当前场景类指令

当前已经覆盖停止、跳转、调用、重绘、等待、事件方向/位置/状态、队伍方向/位置/成员、日夜调色板、音乐和音效请求、角色 Sprite 切换、场景切换、队伍轨迹移动以及带 RGM 肖像的对话消息。战斗、物品、装备、商店、数值判断等指令会在 M3–M4 按系统逐步补齐。

### 已实现操作码速查

| 操作码 | 含义 | 主要操作数 |
| --- | --- | --- |
| `0000`–`0004` | 停止、替换未来入口、跳转、调用子脚本 | `op0` 入口，`op1` 事件编号 |
| `0005`、`0008`、`0009` | 重绘、保存下一入口、等待脚本帧 | `op0` 帧数，`op1` 重绘延迟 |
| `000B`–`0011` | 自动脚本 NPC 单步或走到目标 | `op0/op1` 格坐标，`op2` half |
| `000F`、`0012`–`0016` | 设置事件/队员方向、动作和位置 | 事件编号、方向、帧或世界偏移 |
| `001E` | 增减金钱，不足时跳转 | signed `op0` 金额，`op1` 失败入口 |
| `001F`、`0020` | 增加或移除物品 | `op0` 物品，`op1` 数量，`op2` 失败入口 |
| `0024`、`0025`、`0040` | 修改事件自动脚本、触发脚本和触发模式 | `op0` 事件，`op1` 新值 |
| `003B`–`003E`、`FFFF` | 开始不同位置的对话或 Toast，输出消息 | 肖像、颜色、M.MSG 索引 |
| `0043`、`0045`、`0047` | 背景音乐、战斗音乐和音效 | `op0` 资源编号；`0043 op1` 控制循环/淡入淡出 |
| `0046` | 设置队伍地图位置 | `op0/op1` 格坐标，`op2` half |
| `0049`、`0052` | 设置或暂时隐藏事件 | 事件、状态或隐藏帧数 |
| `0050`、`0073` | 场景淡出/渐变占位 | 完整视觉渐变仍待实现 |
| `0053`、`0054` | 切换日间或夜间调色板 | 无 |
| `0059` | 切换剧情场景 | `op0` 为 1-based 场景编号 |
| `0065` | 修改角色普通场景 Sprite | `op0` 角色，`op1` Sprite |
| `006C`、`006E` | 事件或队伍移动一步 | signed 世界偏移和逻辑层 |
| `006F` | 按另一事件状态同步当前事件 | 事件编号和目标状态 |
| `0070`、`007A`、`007B` | 队伍以 2/4/8 速度逐帧走到目标 | `op0/op1` 格坐标，`op2` half |
| `0075` | 重建最多三人的队伍 | 三个 1-based 角色编号 |
| `007D`、`007E` | 移动事件或修改事件逻辑层 | 事件、偏移或层级 |
| `0081` | 未面向/接近事件时跳转 | 事件、距离级别、失败入口 |
| `0085`、`008E` | 延迟及对话换页/恢复场景 | `op0×80ms` 或无 |

`ScriptVM` 源码在每个 `match entry.operation` 分支前保留同样的中文说明。主触发脚本、逐帧自动脚本和即时安全子脚本虽然共用操作码，但等待能力不同：自动脚本不能打开会阻塞玩家输入的 UI，即时子脚本只接受不会暂停的安全子集。

音乐请求不会按地图编号推测。`0043` 把曲目、循环和淡入淡出参数交给 `PalAudioPlayer`；`0045` 只更新后续战斗曲目；`0047` 保留原始 VOC 编号。离线转换和运行时声道详见[音乐、音效与音量](AUDIO.md)。

## 回归测试

不依赖原版数据的合成测试覆盖消息暂停、单步继续和脚本结束。合法取得并导入本地资源后，可运行首场景集成测试：

```bash
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . \
  --script res://tests/run_local_intro_test.gd
```

测试只记录指令是否完成和消息数量，不输出或提交原版对话内容。当前资源的首场景进入脚本可以执行到结束，不含未支持指令。

全部剧情场景的自动事件回归：

```bash
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . \
  --script res://tests/run_local_event_auto_script_test.gd
```

该测试让 294 个场景的 EventObject 自动脚本各运行 120 帧，检查 NPC 动作、状态和入口发生变化，同时禁止直接指令或即时嵌套子脚本静默停在未支持操作。只需快速检查前六个场景时可追加 `-- --early-scenes`。

日常画面验证可从资源实验室进入“剧情测试”。人工入口只保留桂花酒流程相关的待验收问题；已确认完成的开场、厨房入口、对话和楼梯检查点会从界面移除，继续由本地自动回归测试覆盖。该入口仅修改本次运行的临时会话，不读写正式存档。
