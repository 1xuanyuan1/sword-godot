# 脚本虚拟机与对话流程

Godot 版 `ScriptVM` 以固定 SDLPal 基准的 `script.c` 为行为参考。当前 DOS 数据实际使用的 164 种操作码，已经在主触发、EventObject 自动、即时子脚本、装备、战斗效果和敌人战斗六类真实运行上下文中全部接通；支持结论由真实脚本根入口控制流审计产生，不再以“任意解释器里存在同名 `case`”代替运行时覆盖。

## 执行模型

- 解释器保存当前指令、事件对象编号和脚本调用栈。
- `0004` 调用的子脚本结束后会返回父脚本继续执行。
- 主触发、自动和即时解释器共用 `_interpret_common_instruction()` 非阻塞执行层。统一结果包含 `handled`、`flow`（前进、原地重试、跳转）、`jump_entry`、`script_failed` 和 `world_changed`；调度器只负责应用游标、等待和逐帧刷新语义。NPC 移动、事件状态、属性判断、调色板、跟随者和动态地图等公共指令不再维护三份容易分叉的实现。
- 对话、等待、战斗、`000A` 确认、`0026/0027` 商店、`004D` 任意键和 `004E` 读档仍由主 VM 的阻塞状态机调度。正文必须先完成当前等待轮次，才会打开后续确认框或商店。
- `FFFF` 消息中的“角色名：”会识别为说话人标题，不单独等待；紧随其后的正文显示后，解释器暂停，玩家按空格或回车继续。
- 连续 `FFFF` 正文会合并为同一轮显示；下一条只要不是消息，VM 就先等待本轮对话确认，坐标、事件状态、乘船和 `0059` 场景切换都不能在对白结束前提前执行。
- 上、下、中部对话位置由 `003B`–`003E` 设置；脚本结束或重绘时关闭当前对话框。
- `003C`、`003D` 传入的 `RGM.MKF` 肖像编号会显示在左侧角色栏，姓名位于头像下方，正文位于右侧；无肖像旁白自动扩展正文区域。
- 对于脚本省略肖像、但保留“角色名：”的台词，内容数据库会从完整脚本统计该角色最常使用的肖像作为回退；明确的无标题角色台词使用单独元数据，避免误伤旁白。
- M.MSG 正文中的 `$NN` 与 `~NN` 按官方 `TEXT_DisplayText` 解释为逐字速度和句末停顿，不会把 `$06`、`$04` 或 `~30` 等控制码画到对话框中。
- `003E` 系统消息使用中央黑底白字 Toast，约 1.4 秒后自动关闭，不需要按键。
- `003B` 普通中央对白保持交互式显示；以成对引号标记的无角色剧情叙述复用系统 Toast，连续消息会先合并，再统一计时关闭。
- 触发事件时传入全局 `EventObject` 编号，使方向、帧、位置、状态和层级修改作用于正确对象。
- 搜索事件使用空格/回车；它不是“寻找像素距离最近的对象”，而是按人物朝向依次扫描 13 个 PAL half 格检查点。`trigger_mode 1/2/3` 分别只允许前 2/8/13 个点，同一检查点按 EventObject 全局顺序选择第一个。
- 搜索命中后，处于普通四方向动画范围内的 NPC 会恢复站立帧并转向队伍；队伍也会清除上一段剧情留下的强制动作。特殊剧情帧不会被搜索覆盖。
- 门、楼梯和传送点按照 `trigger_mode 4–8` 在接近时自动触发。加权距离固定为 `abs(dx) + abs(dy) × 2`，并使用严格小于 16/48/80/112/144 的边界。
- 接触有动画的 NPC 时，NPC 会恢复第 0 帧并按相对位置转向，队伍恢复站立姿势。多个范围重叠的接触事件按 EventObject 顺序执行；因为 Godot VM 可异步等待对话，`MapExplorer` 会保存扫描索引，待当前脚本真正结束后继续下一个对象。

## EventObject 自动脚本

场景事件以 10 FPS 调用 `PAL_RunAutoScript` 兼容层。普通指令每帧执行一条；`0003` 无条件跳转和 `0006` 概率跳转按官方 `goto begin` 在同一帧继续目标指令，避免 NPC 路线无故多停一帧。

当前自动脚本已支持等待、概率分支、子脚本、四方向单步、不同速度走到目标、方向/动作、修改目标脚本与状态、剧情音效、临时隐藏、追逐队伍、直接移动、逻辑层、区域判断和原地动画。追逐使用当前 `PalMapData` 与 EventObject 阻挡，浮空事件按脚本参数忽略阻挡。

事件消失状态对齐 `play.c`：负 `vanish_time` 计到零后直接重现；`0052` 会先把状态变负，正计时归零后仍保持隐藏，直到对象离开 SDLPal 的 320×320 镜头保护区才恢复正状态和第 0 帧。自动移动的 NPC 进入接触范围时，同一更新周期会触发其接触脚本。

阻挡型 NPC 的自动路线若挤到队伍脚下，`MapExplorer` 会从 `(NPC 方向 + 1) % 4` 开始按 PAL 方向枚举尝试四个相邻 half 格，把队伍平移到第一个未阻挡位置。该位移不写入主动行走轨迹，也不改变队伍朝向；这用于避免门口 NPC 把玩家永久卡住。

## 当前场景类指令

当前已经覆盖停止、跳转、调用、重绘、等待、事件方向/位置/状态、队伍方向/位置/成员、角色属性与成长、物品/装备/满血/场景判断、商店、存读档、调色板与画面效果、RNG/FBP/结局、经典战斗等待与胜败分支、跟随者和动态换图。装备菜单中的 `0017/0018/001A/002D` 仍由带部位上下文的 `PalEquipmentManager` 解释；普通剧情与场外仙术中的 `002D` 由 ScriptVM 写入角色临时状态。战斗专用数值和演出留在 `PalBattleController`，不会把战斗临时状态泄漏到探索 VM。

`0084` 只查询正式 `PalMapData` 阻挡，并按队伍方向把目标 EventObject 放到前方 half 格；失败分支不会修改物品或事件状态。`0098` 的两名跟随者分别使用轨迹第 4、5 格，并与队伍、事件共同参与 `PalTileMapWorld` 的 Y 排序和遮挡。`0099` 只修改目标场景的 `map_number`；当前场景会重载 `TileMapLayer`，但不会重复执行场景进入脚本，运行时地图编号继续随版本化存档保存。

`DATA.MKF #0` 由 `PalStoreDefinition` 按每店九个物品解析。购买页检查现金和 99 个持有上限；出售页只显示可出售物品并按基础价格一半结算。两种页面复用经典菜单字体、边框、键盘导航和当前调色板。

`GameSession` 统一提供 PLAYERROLES 字段修改、仙术移除、剧情升级/随机成长、物品与装备数量、满血判断、跟随者及收妖值等领域操作。`palette_index`、跟随者、收妖值、追逐范围以及场景动态地图均进入存档；`auto_battle_pending` 只影响紧随其后的战斗，控制器建立战斗后立即消费，读档时强制清空。

`ScriptVM` 源码在每个 `match entry.operation` 分支前保留同样的中文说明。主触发脚本、逐帧自动脚本和即时安全子脚本共享公共非阻塞执行层，但等待能力仍不同：自动脚本不能打开会阻塞玩家输入的 UI，即时子脚本只接受不会暂停的安全子集。

`000B`–`000E` 不只用于自动巡逻，也会出现在乘船等主触发剧情中：当前 EventObject 按南、西、北、东设置方向，以速度 2 前进一步并推进动作帧，夹在其后的 `0005` 负责把每一步刷新到屏幕。`003B` 的引号叙述使用 Toast；`003C/003D` 在肖像编号为 0、下一条消息同样以成对引号开始时也改用 Toast，避免道具获得和行为描述误占角色对话框。

`0015` 指定的队员动作帧优先于渲染器残留的步态标志。正常 `record_party_step()` 会先清除旧剧情姿势，因此玩家移动时仍会恢复行走动画；如果同一段脚本在 `006E` 小步移动后再次执行 `0015`，新姿势必须立即覆盖步态。仙灵岛追打剧情会把李逍遥切到特殊 Sprite 193，再指定第 0 帧表现倒地，不能继续显示刚才移动产生的直立帧。

`003F`、`0044`、`0097` 对齐官方 `PAL_PartyRideEventObject`：每个脚本帧用同一位移更新队伍视口、轨迹和当前 EventObject，并保留登船时通过 `0015` 设置的固定动作。余杭乘船脚本 `0x1725` 使用低速 `003F` 驶离码头，到达目标后继续执行队伍落点和 `0059`，进入场景 15（运行时索引 14）的仙灵岛区域。

`0050`、`0051` 对齐官方 `PAL_FadeOut` / `PAL_FadeIn`，动画期间 VM 与玩家输入都暂停。`0050` 完成后保留“下一次画面绘制时渐显”的状态：普通 `0005` 重绘和 `007F` 镜头移动都对应官方的 `PAL_MakeScene`，RNG 则由 `PAL_RNGPlay` 在第一帧更新后消费；必须在新地图、人物、镜头或电影首帧真正准备好后才揭开黑色遮罩。乘船流程因此先在余杭驶离并渐隐，切到仙灵岛码头后由进入脚本的 `0005` 重绘触发渐显；仙灵岛洗澡脚本 `9649` 则在花树场景 13（地图 119）连续使用 `0050 → 007F`，分别切换发现衣服、赵灵儿洗澡和李逍遥用树枝晃衣服的镜头。若只等待后面的 `0005`，黑色遮罩会覆盖本应可见的过场；若在画面准备前过早渐显，又会露出旧内容。官方目的地脚本没有第二条乘船位移指令，抵达段的视觉边界来自渐隐与渐显。

`0076` 对齐 `PAL_ShowFBP`，显示导入自 `FBP.MKF` 的 320×200 索引图片；`FFFF` 按官方解压失败路径得到全黑画面。FBP 层位于地图和普通 HUD 上方、剧情对话框下方，因此黑屏期间仍可显示叙述；如果此前 `0050` 已留下不透明的顶层渐隐遮罩，黑色 FBP 就在准备完成后替换该遮罩，避免“一夜过去”被盖住。`op1 > 0` 时 VM 会等到渐显完成再续跑。`0093` 对齐 `PAL_SceneFade`，按 signed 步长决定方向和时长；夜间客栈剧情使用 `0093 FFFE → 0076 FFFF → 0051 FFFF` 完成约 3.2 秒场景渐隐、黑屏和默认速度渐显。这里的 `0051 FFFF` 必须先转为 signed SHORT，不能误当成 65535 倍速度。

音乐请求不会按地图编号推测。`0043` 把曲目、循环和淡入淡出参数交给 `PalAudioPlayer`；`0045` 只更新后续战斗曲目；`0047` 保留原始 VOC 编号。`00A3` 原本优先播放 CD 音轨，Godot 桌面版按官方失败回退路径循环播放 `op1` 的 RIX 曲目；导入器也会扫描这个回退编号。离线转换和运行时声道详见[音乐、音效与音量](AUDIO.md)。

`007F` 的镜头位置独立于队伍世界坐标：普通形式每个 10 FPS 脚本帧累加 signed `op0/op1`，`op2=FFFF` 把镜头固定到 PAL 格坐标，两个零操作数恢复普通队伍跟随。`MapExplorer` 将偏移同时交给 TileMap Camera2D 和 CPU 对照渲染器；人物碰撞、事件触发、轨迹和存档位置均不随镜头平移。`0059` 切场景时复位临时偏移。

`0080` 对齐官方 `script.c` 的昼夜调色板切换。水月宫成亲脚本在求婚对白后用它进入夜间，随后渐隐、显示“一夜过去”，再以 `0053` 恢复白天；如果解释器把它当作未支持指令，脚本会在床边提前结束并错误恢复玩家控制。Godot 当前在 `op0=0` 时同步最终调色板画面，逐色调色板渐变仍属于后续视觉精度优化。

`0038` 不会在每次 `0059` 切场景时自动运行。它只在原版脚本明确要求“传送离开当前场景”时调用当前 `SCENE.script_on_teleport`，子脚本可以设置队伍落点、发出 `0059` 场景切换和淡出；完成后返回调用者继续音效或 `00A1` 队伍收拢。场景没有离开脚本时，VM 标记失败并跳到 `0038 op0`。

`0037` 与对话一样是显式等待边界。`MapExplorer` 在 HUD 播放器完成后调用 `complete_rng_animation()`；这之前不能移动、开菜单、触发接触事件或执行动画之后的 `0055`。如果前一条 `0050` 留下待渐显状态，`PalRngPlayer` 会保持第 0 帧可见但暂停计时，顶层黑色遮罩渐显到透明后才恢复原脚本帧率，复现官方 `PAL_RNGPlay()` 的首帧时序。本地帧缺失时播放器也会异步完成，保证测试和损坏资源诊断不会让 VM 永久停住。

山神庙御剑教学是这组等待和状态指令的真实主线验收：EventObject 196 从入口 `6622` 执行消息 `1360–1400`，停止原场景音乐、以 `0050 → 0036/0037` 播放 RNG #1 的完整区间，再用 `0055` 让李逍遥习得对象 345、用 `001D` 恢复全队 HP/MP、用 `0053` 切回日间调色板。结尾还会批量改写后续 EventObject，并以 `006D` 把场景 7 的进入脚本安装为 `6767`；任何一个等待边界提前续跑，都会让技能或场景状态在动画完成前生效。带窗口回归还会检查渐显后的 320×200 实际像素并保存到被忽略的 `generated/pal/visual_tests/training_rng_001.png`，不能只断言 RNG 请求编号正确。

客房赵灵儿营救脚本 `6906–7017` 是战斗等待之后继续解释场景指令的真实验收。战前用 `0075` 临时组成李逍遥／赵灵儿队伍，并由 `004A/0007` 请求敌队 18、战场 21；胜利后先经过 SDLPal `script.c` 明确保留为空操作的 `0078`，再用 `0022` 复活倒下队员并清除三级以下毒和临时状态，最后用 `001D` 回满 HP/MP、正式保留赵灵儿并更新后续事件。`0078` 不能当作未支持指令提前终止，否则战斗虽然能打开，战后剧情不会发生。

`0007` 进入独立的 `waiting_for_battle`。VM 发出敌队、`004A` 战场和 `operand[2] == 0` 推导的 Boss 标志；覆盖层确认结果后，胜利继续下一条，战败跳 `op1`，逃跑跳 `op2`。等待期间新的触发脚本会被拒绝，`stop()` 和脚本结束也会清空暂存分支。

`006D` 直接修改本次运行中的 `PalSceneDefinition`。非零 `op1/op2` 只覆盖对应入口，另一个入口保持不变；两者都为零时同时清空。它用于求药归来后把场景 1 的进入脚本改为夜间山神庙提醒。`009A` 使用包含首尾的全局 EventObject 编号范围，状态按 16 位有符号数解释；这与单对象 `0049` 的状态语义一致。
